- 修了一个本博客的Bug

如下图所示，我博客是基于`NextJS`的，打包成静态文件中有一个个的 `*.html` 文件
![NextjsPKG](/static/images/NextjsPKG.png)

错误的Nginx配置：
```
server {
     listen 80;
     root /apps/blog/build;
     server_name blogs.runsunway.com;
     location / {
        index index.html;
        try_files $uri /$uri /index.html;
     }
}
```

正确的Nginx配置：

```
server {
     listen 80;
     root /apps/blog/build;
     server_name blogs.runsunway.com;
     location / {
        index index.html;
        try_files $uri $uri.html /$uri /index.html;
     }
}
```

`try_files` 配置多加一个`$uri.html`以便于直接通过`URL`访问特定的博客

比如直接访问这篇博客: https://blogs.runsunway.com/blog/Software-Based-Router

如果不加`$uri.html`则无法匹配在`root /apps/blog/build`下匹配正确的`URI`，会根据`try_files /index.html`规则直接跳转 `index.html 首页`

不加`$uri.html`就只能在`URL`后补全`.html`：https://blogs.runsunway.com/blog/Software-Based-Router.html
